# Failure And Readiness

This folder groups docs that make incompleteness visible.

## Purpose

- Make known weaknesses visible.
- Separate installability from implementation readiness.
- Prevent placeholder docs from sounding finished.
- Prevent catalog entries from implying real behavior.
- Prevent candidate kits from being treated as official.

Primary docs live at the repo root and under `docs/` so they are easy to find.
